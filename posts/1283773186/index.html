配置管理中心secret

Secret 是什么？

上面我们学习的 Configmap 一般是用来存放明文数据的，如配置文件，对于一些敏感数据，如密码、私钥等数据时，要用 secret 类型。

Secret 解决了密码、token、秘钥等敏感数据的配置问题，而不需要把这些敏感数据暴露到镜像或者 Pod Spec 中。Secret 可以以 Volume 或者环境变量的方式使用。

要使用 secret，pod 需要引用 secret。Pod 可以用两种方式使用 secret：作为volume 中的文件被挂载到 pod 中的一个或者多个容器里，或者当 kubelet 为 pod 拉取镜像时使用。

secret 可选参数有三种

  • generic: 通用类型，通常用于存储密码数据。
  • tls：此类型仅用于存储私钥和证书。
  • docker-registry: 若要保存 docker 仓库的认证信息的话，就必须使用此种类型来创建。

Secret 类型

Service Account：用于被 serviceaccount 引用。serviceaccout 创建时Kubernetes 会默认创建对应的 secret。Pod 如果使用了 serviceaccount，对应的secret 会自动挂载到 Pod 的 /run/secrets/kubernetes.io/serviceaccount 目录中。

Opaque：base64 编码格式的 Secret，用来存储密码、秘钥等。可以通过 base64 – decode 解码获得原始数据，因此安全性弱

kubernetes.io/dockerconfigjson：用来存储私有 docker registry 的认证信息。

使用 Secret

通过环境变量引入 Secret
把 mysql 的 root 用户的 password 创建成 secret

kubectl create secret generic mysql-password –from-literal=password=123123

查看

kubectl get secret

详细信息

kubectl describe secret mysql-password

创建 pod，引用 secret
apiVersion: v1 # Kubernetes API的版本号  
kind: Pod # 资源的类型为Pod  
metadata:  
  name: pod-secret # Pod的名称是pod-secret  
  labels: # Pod的标签  
    app: myapp # Pod的标签键为app，值为myapp  
spec: # Pod的规格定义  
  containers: # 定义Pod中的容器列表  
  - name: myapp # 容器的名称是myapp  
    image: ikubernetes/myapp:v1 # 容器使用的镜像名称和标签  
    ports: # 定义容器暴露的端口  
    - name: http # 端口的名称是http  
      containerPort: 80 # 容器内部监听的端口号是80  
    env: # 定义容器的环境变量  
    - name: MYSQL_ROOT_PASSWORD # 环境变量的名称是MYSQL_ROOT_PASSWORD  
      valueFrom: # 环境变量的值来源于一个引用  
        secretKeyRef: # 引用的是Secret资源中的某个key的值  
          name: mysql-password # Secret资源的名称是mysql-password  
          key: password # Secret资源中用于获取值的key名是password
通过 volume 挂载 Secret
创建 Secret
apiVersion: v1 # Kubernetes API的版本号  
kind: Secret # 资源类型为Secret  
metadata:  
  name: mysecret # Secret的名称是mysecret  
type: Opaque # Secret的类型是Opaque，表示这是一个通用的Secret，不特定于任何特定的程序或用途  
data: # data字段用于存储Secret的数据，其值应该是Base64编码的  
  username: YWRtaW4= # 用户名字段，其值为Base64编码的字符串，解码后为"admin"  
  password: MTIzMTIz # 密码字段，其值为Base64编码的字符串，解码后为"123123"

kubectl apply -f secret.yaml

kubectl describe secret mysecret

将 Secret 挂载到 Volume 中
apiVersion: v1 # Kubernetes API的版本号  
kind: Pod # 资源的类型为Pod  
metadata:  
  name: pod-secret-volume # Pod的名称是pod-secret-volume  
spec: # Pod的规格定义  
  containers: # 定义Pod中的容器列表  
  - name: myapp # 容器的名称是myapp  
    image: registry.cn-beijing.aliyuncs.com/google_registry/myapp:v1 # 容器使用的镜像名称和标签，从阿里云的北京区域仓库获取  
    volumeMounts: # 定义容器中的卷挂载  
    - name: secret-volume # 挂载的卷的名称是secret-volume  
      mountPath: /etc/secret # 卷在容器中的挂载路径是/etc/secret  
      readOnly: true # 挂载的卷是只读的  
  volumes: # 定义Pod中的卷列表  
  - name: secret-volume # 卷的名称是secret-volume  
    secret: # 定义卷类型为Secret  
      secretName: mysecret # 使用的Secret资源的名称是mysecret
