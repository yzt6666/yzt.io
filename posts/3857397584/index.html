Docker容器引擎基础

发表时间：2024-02-21 阅读： 评论：

Docker

容器背景

随着云计算和DevOps生态圈的蓬勃发展，出现的问题是需要维护一个非常庞大的开发、测试、预上线和生产环境。

虚拟化的几个缺点

  1. 资源占用多。
  2. 冗余步骤多。
  3. 启动慢。

容器技术的优势

  1. 启动快。
  2. 资源占用少。
  3. 体积小。
  4. 简化部署。

Docker镜像操作

docker search 关键字 ：从Docker Hub搜索镜像。

docker pull 镜像名称:[标签]：获取镜像，从网络上下载。

docker images 仓库名称:[标签]：查看镜像信息。

docker inspect 镜像名或镜像ID：获取镜像的详细信息。

docker tag 名称[:标签] 新名称[:新标签]：为镜像添加新的标签。

docker rmi 镜像名称[:标签]：删除镜像。

docker rmi 镜像ID：删除镜像。

docker save -o 存出镜像为本地文件名 镜像名：存出镜像。

docker load < 存出的文件：载入镜像。

docker load –input 存出的文件：载入镜像。

docker push 仓库名称:标签：上传镜像。

Docker容器操作

1、创建容器与运行容器

命令格式：docker create [选项]… 镜像运行的程序

常用选项：

-i 让容器的输入保持打开

-t 让Docker 分配一个伪终端

-d 守护进程形式运行

例如：docker create -it centos /bin/bash

可以使用docker ps 命令来查看所有容器的运行状态。添加-a 选项可以列出系统中所有容器状态。

2、容器的启动与停止

命令格式：docker start 容器的ID/名称

如果用户想创建并启动容器，可以直接执行docker run 命令，等同于先执行docker create 命令，再执行docker start 命令。

创建容器并启动执行一条shell 命令：docker run centos /bin/bash -c ls /

docker run -d：容器在守护态形式在后台进行运行。

下面的容器会持续在后台进行运行：

[root@localhost ~]# docker run -d nginx /bin/bash -c “while true;do echo hello;done”

fa286930dde0ac3dde6750812929e42c650bb78a2d6e96cf1373a872266a1bc2

查看容器内的输出：

[root@localhost ~]# docker logs fa286930dde0

2、容器的运行与终止

命令格式：docker stop 容器的ID/名称

3、容器的进入

命令格式：docker exec -it 容器ID/名称/bin/bash

-i ：表示让容器的输入保持打开；

-t ：表示让Docker 分配一个伪终端。

通过exit 命令退出容器。

通过Ctrl+pq命令退出容器并保证容器在后台继续运行。

4、容器的导出与导入

导出容器到文件

命令格式：docker export 容器ID/名称>文件名

导入文件成为本地镜像

命令格式：cat 文件名| docker import – 生成的镜像名称:标签

5、容器的删除

命令格式：docker rm 容器ID/名称

6、文件复制

将主机/root/anaconda-ks.cfg目录拷贝到容器1708ad171a07的/tmp目录下

docker cp /root/anaconda-ks.cfg 1708ad171a07:/tmp

docker attach 1708ad171a07：进入容器

创建私有仓库

通过官方提供的registry镜像部署一套本地的私有仓库环境

[root@localhost ~]# mkdir -p /opt/data/registry
[root@localhost ~]# docker run -d -p 5000:5000 -v /opt/data/registry:/tmp/registry registry

准备测试镜像

docker run -d -p 8000:80 nginx //将宿主机8000端口映射给容器的业务端口

将修改过的nginx镜像做标记封装，准备上传到私有仓库

docker tag nginx 192.168.200.101:5000/nginx-test

上传之前设置docker客户端使用http协议

[root@localhost ~]# cat /etc/docker/daemon.json

{“insecure-registries”:[“192.168.200.101:5000”]}

将镜像上传到私有仓库

[root@localhost ~]# docker push 192.168.200.101:5000/nginx-test

从私有仓库中下载镜像到本地

[root@localhost ~]# docker pull 192.168.200.101:5000/nginx-test
